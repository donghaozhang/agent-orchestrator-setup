# Create Implementation Plan

Create a detailed implementation plan as a markdown file.

## Steps

1. Analyze the feature/task provided by the user.
2. Create a markdown plan file with:
   - Overview of the feature
   - Architecture decisions and trade-offs
   - Step-by-step implementation tasks
   - Relevant file paths for each task
   - Unit test plan with file paths
3. If implementation would take longer than 20 minutes, break into subtasks.
4. Each subtask should include:
   - Relevant file paths
   - Acceptance criteria
   - Dependencies on other subtasks
5. Prioritize long-term maintainability over short-term shortcuts.
6. Save the plan to `docs/plans/<feature-name>.md`.
